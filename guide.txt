# Deployment Guide: Wordle-Duel

This guide outlines how to deploy the Vite frontend to Vercel and provides suggestions for deploying the Go WebSocket backend.

## 1. Deploying the Frontend (Vercel)

Vercel is an excellent choice for deploying Vite applications.

### Prerequisites:
- Your project should be pushed to a Git repository (GitHub, GitLab, or Bitbucket).

### Steps:
1. Log in to your [Vercel Dashboard](https://vercel.com/dashboard).
2. Click **Add New...** and select **Project**.
3. Import your Wordle-Duel repository.
4. Configure the project settings:
   - **Framework Preset**: Vercel should automatically detect **Vite**. If not, select it.
   - **Root Directory**: Click `Edit` and select `Frontend`.
   - **Build Command**: `npm run build` (default for Vite).
   - **Output Directory**: `dist` (default for Vite).
5. **Environment Variables**:
   In the Environment Variables section, add the URLs for your deployed backend:
   - `VITE_API_URL`: `https://your-deployed-backend-url.com`
   - `VITE_WS_URL`: `wss://your-deployed-backend-url.com/ws`
   *(You will get these URLs after deploying your backend in step 2)*
6. Click **Deploy**. Vercel will build and host your frontend.

---

## 2. Deploying the Backend (Suggestions)

Since the backend is built with Go and heavily relies on WebSockets for real-time multiplayer, **Vercel is not recommended for the backend**. Serverless platforms like Vercel have strict execution timeouts and do not support persistent WebSocket connections effectively.

Instead, you need a platform that supports long-running processes and WebSockets. Since you already have a `Dockerfile` in the `Backend` directory, deployment will be straightforward on container-friendly platforms.

Here are the best options for deploying your Go backend:

### Option A: Render (Recommended)
Render natively supports Docker and WebSockets with zero extra configuration.
1. Create a [Render](https://render.com/) account.
2. Click **New +** and select **Web Service**.
3. Connect your Git repository.
4. Configuration:
   - **Root Directory**: `Backend`
   - **Environment**: Docker (Render will automatically detect your `Dockerfile`).
5. Click **Create Web Service**. Render will build the Docker container and provide you with a live URL (e.g., `https://wordle-backend.onrender.com`).

### Option B: Railway
Railway is another excellent platform with a great developer experience.
1. Log in to [Railway](https://railway.app/).
2. Click **New Project** -> **Deploy from GitHub repo**.
3. Select your repository.
4. Go to the service settings, and set the **Root Directory** to `/Backend`.
5. Railway will detect the `Dockerfile` and deploy the service. It supports WebSockets out of the box.

### Option C: Fly.io
Fly.io runs your Docker containers close to your users. It's fantastic for multiplayer games to reduce latency.
1. Install the `flyctl` CLI.
2. Open a terminal in your `Backend` directory.
3. Run `fly launch` and follow the prompts. It will detect the `Dockerfile` and generate a `fly.toml` file.
4. Run `fly deploy` to push the app live.

### Next Steps After Backend Deployment:
Once your backend is live, take note of its URL (e.g., `https://my-go-backend.app`). 
1. Go back to your Vercel frontend project settings.
2. Update the `VITE_API_URL` to `https://my-go-backend.app`.
3. Update the `VITE_WS_URL` to `wss://my-go-backend.app/ws`.
4. Trigger a new deployment on Vercel so the frontend uses the production backend URLs instead of `localhost`.
